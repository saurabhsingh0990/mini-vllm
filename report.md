# 📊 Mini vLLM Project Report (Decoding Module)

---

## 📌 1. Introduction

This project aims to build a **production-style LLM inference engine** with a focus on understanding and implementing core system-level components.

This report currently covers the **Decoding Module**, which is responsible for controlling how tokens are generated from a language model.

---

## 🎯 2. Problem Statement

Default high-level APIs (like HuggingFace `.generate()`) abstract away the internal decoding process, limiting:

* transparency in token generation
* control over output quality
* ability to optimize inference

This module aims to **replace the black-box generation pipeline** with a custom implementation.

---

## 🧠 3. Decoding in LLMs

Decoding refers to the process of **selecting the next token** from a probability distribution generated by the model.

At each step:

1. Model outputs logits for all tokens
2. Logits are converted to probabilities
3. A decoding strategy selects the next token

---

## ⚙️ 4. Implemented Decoding Strategies

---

### 4.1 Greedy Decoding

**Approach:**

* Select token with highest probability at each step

**Advantages:**

* Simple and fast
* Deterministic

**Limitations:**

* Leads to repetitive outputs
* Suffers from **mode collapse**
* Low diversity

---

### 4.2 Top-K Sampling

**Approach:**

* Select top K most probable tokens
* Sample randomly from these tokens

**Advantages:**

* Introduces diversity
* Reduces repetition

**Limitations:**

* Fixed K may not adapt to distribution
* Can still produce suboptimal outputs

---

### 4.3 Top-P (Nucleus) Sampling

**Approach:**

* Select smallest set of tokens whose cumulative probability ≥ p
* Sample from this dynamic set

**Advantages:**

* Adaptive token selection
* More natural and coherent output
* Better balance between randomness and quality

## 🧠 6. Key Learnings

* Greedy decoding leads to **degenerate text generation**
* Sampling methods improve **diversity and realism**
* Top-P sampling provides **adaptive and superior performance**
* Token-level control is essential for building efficient inference systems
